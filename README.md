# 🐙 Feed the Kraken Online - Backend

محرك لعبة كامل للنسخة الرقمية من لعبة Feed the Kraken، مع راوٍ بشري يتحكم بالقرارات والنظام يدير اللوجستيات.

## التشغيل

```bash
npm install
npm start        # السيرفر على المنفذ 3000
npm test         # اختبار محاكاة (يلعب 210 لعبة كاملة آلياً)
```

## أوضاع اللعب

عند إنشاء غرفة من لوحة الراوي تختار أحد ثلاثة أوضاع:

- **رحلة سريعة (5-7):** خريطة 13 مساحة، مجموعة 19 ورقة.
- **رحلة طويلة (7-11):** خريطة 23 مساحة، مجموعة 23 ورقة، فيها خط الإمداد.
- **اختبار 3 لاعبين (غير قانوني):** وضع تجريبي للاختبار المحلي السريع — ليس اللعبة الحقيقية.
  بحّار واحد + قرصان واحد + قائد طائفة واحد، على خريطة الرحلة الطويلة. لا تجمّع للقراصنة
  (قرصان واحد يعرف نفسه)، عتبة المطيانة = 2، ولا علامات Off-Duty (القبطان يعيّن الاثنين الآخرين دائماً).
  هذا الوضع بشري فقط ولا يُستخدم في `test/sim.js`.

## ما الذي يديره النظام آلياً؟

- توزيع الفرق السرية (بما فيها قاعدة الـ 5 لاعبين الخاصة)
- التجمع السري للقراصنة (رقمياً: يرون بعضهم في التطبيق - لا حاجة لإغلاق عيون!)
- توزيع بطاقات الشخصيات + القبطان الأول
- كل مراحل الجولة: تعيين الفريق ← سؤال الولاء ← الملاحة ← Off-Duty
- المطيانة: الالتزام السري، الكشف، العتبات، التعادل، خسارة المسدسات
- الملاحة الكاملة: سحب، اختيار، الـ Logbook، الكشف، الحركة
- رفض الأوامر + ملاحة الطوارئ
- Map Actions: فحص الكابينة (سري)، الجلد (عشوائي صحيح)، قطع اللسان، إطعام الكراكن
- Card Actions: السُكر، حورية البحر، التلسكوب، التسليح، نزع السلاح
- طقوس الطائفة الثلاثة + قواعد التحويل (غير القابلين للتحويل)
- خط الإمداد، إعادة الخلط، الـ résumés، شروط الفوز كلها

## دور الراوي

الراوي عنده `narrator:override` للتدخل بأي شيء:
`set_phase`, `set_guns`, `set_captain`, `move_ship`, `eliminate`, `revive`,
`set_off_duty`, `end_game`, `force_finish_round`, `reorder_seats`

ويرى **كل شيء**: الفرق الحقيقية، الأوراق، الالتزامات السرية.

## أحداث Socket.io الرئيسية

| الحدث | من | الوصف |
|---|---|---|
| `narrator:create_room` | راوي | إنشاء غرفة، يرجع كود من 4 أحرف |
| `player:join` | لاعب | `{code, name}` |
| `narrator:start_game` | راوي | بدء اللعبة |
| `narrator:finish_gathering` | راوي | إنهاء التجمع السري |
| `captain:appoint` | قبطان | `{ltId, navId}` |
| `player:commit_guns` | الكل | `{count}` التزام سري |
| `captain:reveal_mutiny` | قبطان | كشف المطيانة |
| `captain:resolve_tie` | قبطان | `{dropId}` حسم التعادل |
| `captain:choose_card` | قبطان | `{keepIndex}` |
| `lt:choose_card` | Lieutenant | `{keepIndex}` |
| `navigator:choose_card` | Navigator | `{discardIndex}` |
| `navigator:deny_command` | Navigator | القفز من السفينة |
| `captain:emergency_navigator` | قبطان | `{navId}` |
| `captain:map_action` | قبطان | `{targetId}` |
| `captain:mermaid` / `captain:telescope_pick` | قبطان | `{targetId}` |
| `player:telescope_decide` | الهدف | `{discard: bool}` |
| `cult:distribute_guns` | قائد الطائفة | `{allocations: {playerId: n}}` |
| `cult:convert` / `cult:skip` | قائد الطائفة | `{targetId}` |
| `character:activate` | الكل | `{targetId?, targetIds?, ...}` |
| `narrator:override` | راوي | `{action, payload}` |

كل حدث يرد بـ `{ok, error?}` عبر acknowledgement، والحالة تُبث تلقائياً:
- `state` - الحالة العامة + `you` (الخاص بك) أو `narrator` (كل شيء)
- `secret` - معلومة سرية لك فقط (نتيجة فحص كابينة، أوراق حورية البحر...)
- `prompt` - طلب قرار منك (عرض المحرّك الخفي)

## الخريطة قابلة للتعديل

`mapLong.js` يبني الخريطة برمجياً (شكل ماسة 1-3-5-5-5-3-1). إذا أردت مطابقة
اللوح الفيزيائي حرفياً، عدّل `ACTION_PLACEMENT` و `computeExits` - كل شيء
data-driven.

## الصفحات (public/)

| الرابط | الوصف |
|---|---|
| `/` | الصفحة الرئيسية - اختيار لاعب / شاشة عرض / راوٍ |
| `/player.html` | **واجهة اللاعب** (موبايل) - خريطة، هوية سرية بكشف مؤقت، كل إجراءات اللعبة |
| `/display.html` | **شاشة العرض المشتركة** (تلفزيون) - اللوح العام بحجم كبير للجلسة |
| `/narrator.html` | **لوحة الراوي** - رؤية كاملة (فرق، أوراق، التزامات سرية)، أدوات تجاوز |

كل الصفحات تحفظ الجلسة وتسترجعها تلقائياً عند إعادة التحميل أو انقطاع الشبكة.

ميزة خصوصية في واجهة اللاعب: الهوية (الفريق + البطاقة + رفاق القراصنة)
مخفية خلف زر "اكشف هويتك" وتختفي تلقائياً بعد 12 ثانية - مناسبة للجلسات
الحضورية حيث الجوالات على الطاولة.
